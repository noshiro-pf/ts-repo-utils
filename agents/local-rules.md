## Project Overview

`ts-repo-utils` is a library of shared repository-automation helpers and CLI
commands used across the author's TypeScript projects Рђћ e.g.
`assert-repo-is-clean`, `format-uncommitted`, `format-diff-from`,
`gen-index-ts`, `check-should-run-type-checks`, plus programmatic helpers
(`$`, `glob`, `formatFiles`, `isDirectlyExecuted`, `projectRootPath`, Рђд).

Uses **pnpm** (see `packageManager`); do not use npm or yarn.

## Self-referential tooling

This repo dogfoods itself: internal scripts (`scripts/cmd/*.mts`) import the
package's own helpers via the package name `ts-repo-utils` (resolved to the
built `dist/` through the `exports` map), the same way `scripts/cmd/build.mts`
already does. Because of this, `pnpm run build` must have produced `dist/`
before scripts that import `ts-repo-utils` can run.

## Generated files (never edit by hand)

- `AGENTS.md` Рђћ generated by `scripts/cmd/gen-agents-md.mts` (run via
  `pnpm run agents:gen`, or automatically during `pnpm run build`) as the
  concatenation of `agents/common-rules.md` + `agents/local-rules.md`.
- `agents/common-rules.md` Рђћ vendored from the common-agent-config repo by
  `pnpm run agents:sync`; edit the shared rules upstream, not here.
- `src/**/index.mts` and index/entry-point files Рђћ generated by `pnpm run gi`.

Regenerate with `pnpm run build`; CI fails if any generated file drifts.

## Workflow

After a series of code changes, run in order: `pnpm run tsc`,
`pnpm run lint:fix`, `pnpm run fmt`, and `pnpm run test`. Run `pnpm run build`
before finishing a task that touches `src/` or the generators.
